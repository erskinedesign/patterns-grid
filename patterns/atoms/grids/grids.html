Grid pattern

A customisable grid system for internal projects

Grid

The .grid class sets a max-width alongside default margins and padding.

.grid

Grid container

The .grid__container simply adds negative horizontal margins so that we don’t need any .last or .first .grid__column classes.

.grid__container

Fluid grid columns

A .grid__column has many modifiers available for sizing the various units within. The width of each is dependent on the number of columns set and the width of the container.

If a .grid__column does not have a modifier such as .grid__column--4 then it naturally takes up the full width of the parent.

Aligning columns

Columns can be aligned to the right with the .grid__column--right class. Due to the way that floats are handled by CSS, sometimes we’ll need to add extra .grid objects in order to clear columns properly.

.grid__column--4
.grid__column--3 .grid__column--right
.grid__column--5
.grid__column--4 .grid__column--right

Responsive columns

Depending on the number of breakpoints in the grid system, you can have any number of combinations of the grid. For instance if a design component needed to be small width at mobile, wide at tablet and then large again at desktop, that is completely possible within this system.

Pushing columns

Columns can be pushed either left or right of the grid container.
