EDX Grid pattern

A customisable, fully responsive grid system with multiple breakpoints.

Features

  • Fluid grid with fixed gutters.
  • Multiple breakpoints for fine control over responsive components.
  • Breakpoint specific grid columns
  • Push columns left or right and center smaller columns.
  • No .clear .start or .end classes necessary.
  • Complimentary grid overlay script.

Compromises

  • Generates an enormous amount of CSS - we recommend using a Grunt tool such as Uncss to strip out any unused styles.
  • Grid units should only be set with these classes in the markup, there’s no ability to change the width on a specific component.
  • Due to the amount of CSS this module currently generates, you can only nest a grid twice.

Custom breakpoints

In this example there are five distinct breakpoints available, although we can add or subtract breakpoints whenever necessary. The width of these breakpoints are entirely dependent on the dimensions of the viewport.

Breakpoint

Grid overlay

With this module comes a complimentary script for displaying the .grid--overlay and automatically generating the number of grid columns. This means that we don’t need to change the script when we change the settings. It’s written in plain ol’ JS so jQuery isn’t a dependency.

Press ‘G’ to activate the grid overlay.

Grid

The .grid class sets a max-width for the breakpoint alongside default margins and padding.

.grid

Grid container

The .grid__container simply adds negative horizontal margins so that we don’t need any .last or .first .grid__column classes.

.grid__container

Fixed gutters

Gutters are fixed width, but can change depending on the breakpoint. In this example the gutter is increased to 25px at Breakpoint D and above.

Grid columns

By default a .grid__column is 100% of the width of the .grid__container. These columns have many modifiers available for sizing the various units within, whilst the width of each is dependent on the number of columns set and the width of the container itself.

Columns can be aligned to the right with the .grid__column--right class. Due to the way that floats are handled by CSS, sometimes we’ll need to add extra .grid objects in order to clear columns properly.

Responsive columns

Depending on the number of breakpoints in the grid system, you can have any number of combinations of the grid, for instance if a design component needed to be small width at mobile, wide at tablet and then small again at desktop. First you’d need to specify the class .grid__column--8--medium followed by .grid__column--6--large. This system allows for an enormous number of design decisions to be made between these breakpoints.

Nesting grids

In order to have nested grids we need to use a nasty hack. First we encapsulate a nested .grid__column with the .grid__subgrid class. Then we use the whacky box model to cheat the child column into thinking that its parent (the subgrid) is actually the width of the full grid__container.

This way we’ll always know that a grid__column--2 will always be 2 columns, regardless of whether it’s nested or not. Due to the crazy amount of CSS this creates you can only nest the grid once.

Pushing a column left or right

Columns can be pushed either left or right of the grid container. Since all columns float left by default, the grid__column--right class must be added in order to float columns to the right.

.push--left-1
.grid__column--right
.push--right-1

Pushing at different viewports

These classes can be modified depending on the viewport, for instance we can push a column to the left on mobile and then to the right on another breakpoint if the design requires it. First we set the width of the column, followed by the direction and how much to push the column by.

.grid__column--10
.push--left-2

.grid__column--5--small
.grid__column--right--small
.push--right-1--small

.grid__column--8--medium
.grid__column--left--medium
.push--left-1--medium

.grid__column--5--large
.grid__column--right--large
.push--right-1--large

Pushing in both directions

Pushing in both directions is possible with the .push--both modifier class. This will always override any of the set widths in the .grid__column classes because in order for this to work the module must be more than 100% wide.

.push--both-1
.push--both-2

Full width cells

If a module is not inside the grid then it will naturally be 100% of the viewport. If possible, all modules should automatically fill up the parent element by default. This way we’ll know precisely how to modify the width of the element – by changing the grid classes in the markup.
